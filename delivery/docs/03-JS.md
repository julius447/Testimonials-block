# 03 — JS

Fil: `ampy-testimonials.js`. Ren vanilla-JS i en IIFE, kräver **Splide v4** (`window.Splide`).
Den **förhöjer** server-renderad markup — bygger inga kort. En enda fil driver godtyckligt
många block på sidan.

## Livscykel
```
DOMContentLoaded → initAll() → för VARJE .ampy-testimonials: initBlock(root)
  initBlock:
    • hämta .att-slider (=.splide), .splide__list, .att-nav (allt via root.*, aldrig document.*)
    • guards: inga kort → göm nav & avbryt · Splide saknas → lämna server-kort (statisk)
    • bygg 4 prickar i .att-nav (en gång)
    • build():   (mount) + körs om vid brytpunkt/reduced-motion-byte
        - perPage 3/2/1, perMove 1, gap 24, type: loop (om antal>perPage annars slide)
        - autoplay 4s (av vid reduced-motion eller om ej loopbart)
        - on 'move'  → uppdatera aktiv prick, nollställ fyllning, starta om cykeln
        - on 'autoplay:playing' → driv fyllningen med Splides VERKLIGA timer-rate (0→1)
    • rAF-watchdog (en gång, för evigt) — se nedan
    • lyssnare: hover-paus, touch-tap-paus, visibilitychange-reset, mq/reduceMQ change → build
```

## Fyllningen (kärnan)
Den aktiva prickens `.att-dot__fill` sätts till `scaleX(rate)`, där `rate` 0→1 kommer från
Splides `autoplay:playing`-event (frame-exakt — kan aldrig drifta mot slide-bytet).
`aktivPrick = splide.index % 4`. Vid `move` nollställs fyllningen och cykeln.
**Verifierat i v4.1.4:** `autoplay:playing`-rate växer 0→1 (fyller vänster→höger).

## Watchdog (självläkande — därför håller det 24/7)
```js
function rafTick() {
  if (splide && autoplayOn && !paused && !reducedMotion && (now - lastPlaying > 900))
    setRate((now - cycleStart) / (INTERVAL + SPEED));   // Splide tyst → driv själv
  requestAnimationFrame(rafTick);
}
```
Primärt driver Splides timer (`lastPlaying` stämplas vid varje event). Om timern tystnar
> 900 ms (t.ex. direkt efter en brytpunkts-rebuild innan nästa event hunnit fyra) tar
rAF-loopen över och håller fyllningen levande. När Splide börjar fyra igen backar loopen
automatiskt. Ingen engångs-flagga som kan låsa fast — självläker över alla rebuilds.

## Antal-säkerhet
* `type: 'loop'` bara om `antal > perPage` (annars `slide` + autoplay av) — loop kräver
  fler kort än som visas.
* Klick på prick `i` → `Math.floor(index/4)*4 + i`, men **avbryts om målet ≥ antalet**
  (ofullständig sista grupp = död prick, ingen out-of-range-navigering).
* Klick på redan aktiv prick startar bara om fyllningen (Splide fyrar då inget `move`).

## Multi-instans
Inga globala id:n. Allt läses via `root.querySelector(...)`. `initAll` kör `initBlock` på
varje `.ampy-testimonials`. Två block på samma sida stör aldrig varandra. IIFE:n hindrar
globalt scope-läckage.

## Paus (WCAG 2.2.2)
* **Mus:** `pauseOnHover` (Splide).
* **Tangentbord:** `pauseOnFocus` (Splide) + fokuserbara prickar.
* **Touch:** tap på ett kort (`pointerdown` med `pointerType==='touch'`) togglar paus via
  `splide.Components.Autoplay.pause()/play()`. (Prickarna ligger utanför slidern → tap på
  prick navigerar, tap på kort pausar.)
* **reduced-motion:** autoplay helt av, aktiv prick står full som statisk markör.

> Vill du ha en **synlig** paus/spela-knapp (striktare AAA-revision) finns mönstret i
> A/B-variant 2:s kod — säg till, så lägger vi in en diskret knapp i nav-raden.

## Konstanter att justera
```js
var INTERVAL = 4000;   // ms mellan byten (tempot ägaren godkänt)
var SPEED    = 500;    // ms glid-transition
var DOTS     = 4;      // antal prickar (cyklar)
```

## Fel-tolerans
Allt Splide-arbete i `try/catch`; saknas Splide eller kort → server-korten står kvar
(inget kraschar sidan). `console.warn` vid mount-fel, aldrig `throw` uppåt.
